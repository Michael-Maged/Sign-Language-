This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

A full-stack assistive communication platform with two directions:
- **Speech → Signs**: React Native app records audio → Whisper transcription → letter breakdown → animated hand-skeleton cards
- **Camera → Letter**: Live camera feed → MediaPipe hand detection → XGBoost classification → letter accumulation into a sentence

## Repository Structure

  main.py               # FastAPI entrypoint — registers speech, sign, media, gesture routers
  routes/

  scripts/

    export_sign_landmarks.py # Step 3: median landmark per letter → data/signs/<LETTER>.json

  data/

    asl_alphabet_train/ # Raw dataset images (not committed — large)

  models/
    hand_landmarker.task     # MediaPipe model binary (must be downloaded separately)

mobile/                 # React Native + Expo app (TypeScript)

  src/
    config.ts           # BASE_URL — local network IP of the backend machine
    screens/
      HomeScreen.tsx         # Speech tab: mic → transcription → letter skeleton cards
      GestureScreen.tsx      # Camera tab: live feed + skeleton overlay + letter accumulation
    components/
      MicButton.tsx          # Record / stop toggle; drives AudioService
      WordRow.tsx            # One word → horizontal scroll of LetterCards

## Backend Commands

```bash
cd backend
source venv/Scripts/activate        # Windows bash; or: venv\Scripts\activate in cmd

# Install dependencies (requirements.txt is empty — install manually)
pip install fastapi uvicorn openai-whisper mediapipe opencv-python xgboost scikit-learn joblib pandas pillow

# Run server
uvicorn main:app --reload           # http://localhost:8000
```

## ML Pipeline (run in order from `backend/`)

```bash
# 1. Extract MediaPipe landmarks from dataset images
python scripts/extract_landmarks.py
# Requires: data/asl_alphabet_train/ and models/hand_landmarker.task
# Output:   data/landmarks.csv  (~4000 rows/letter after flip augmentation)

# 2. Train XGBoost classifier
python scripts/train_model.py
# Output: models/gesture_model.pkl, models/label_encoder.pkl

# 3. Export per-letter landmark JSON for the HandSkeletonView in LetterCard
python scripts/export_sign_landmarks.py
# Output: data/signs/A.json … Z.json
```

## Data Flow

**Speech → Signs (HomeScreen):**
1. `MicButton` → `AudioService.startRecording()` (expo-av)
2. Stop → audio uploaded as `audio/m4a` → `POST /speech-to-text` → Whisper → text
3. Text → `POST /text-to-sign` → `WordModel[]` (word + letters array)
4. Each `LetterCard` fetches `GET /signs/{letter}/landmarks` → renders `HandSkeletonView`
5. `HandSkeletonView` spring-animates 21 joints from wrist origin to sign position

**Camera → Letter (GestureScreen):**
1. `checkServer()` pings `GET /` — shows error screen with diagnosis if unreachable
2. Auto-captures photo every 2.5 s (or on tap) via `CameraView.takePictureAsync()`
3. Photo → `POST /gesture/predict` → `{letter, confidence, landmarks[]}`
4. Raw landmarks (0–1 image coords) mapped to screen coords via cover-mode math and drawn as SVG overlay
5. Pulsing ring animates while scanning; status badge shows live state
6. User taps "Add ✓" (enabled when confidence ≥ 60%) to append letter to sentence

## Key Configuration

- `mobile/src/config.ts` — `BASE_URL` hardcoded to local IP. Must match the machine running the backend. Phone and PC must be on the same Wi-Fi. Run `ipconfig` (Windows) to find your IPv4.
- `gesture.py` loads models at module startup; returns 503 if files are missing.
- `speech.py` loads Whisper `"base"` model at startup (~150 MB download on first run).
- `models/hand_landmarker.task` must be downloaded from MediaPipe and placed in `backend/models/` before running any landmark scripts or the server.
- EXIF rotation is handled in `gesture.py` via `PIL.ImageOps.exif_transpose()` before MediaPipe — required for mobile camera photos.

## Normalization Contract

Both `extract_landmarks.py` (training) and `gesture.py` (inference) use the same `normalize()` function:
- Subtract wrist position (landmark 0) → wrist-relative coords
- Divide by `‖landmark 9‖` (wrist→middle-MCP distance) → scale-invariant

If you change the normalization, update **both** files or the model will produce garbage predictions.
